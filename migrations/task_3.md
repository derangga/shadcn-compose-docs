# Task 3: Install and Configure shadcn/ui (React)

**Priority**: Critical (blocks layout and page tasks)
**Depends on**: Task 2

## Objective

Set up shadcn/ui for React with the same "new-york" style and neutral base color as the current Vue project. Install all UI components needed for the docs layout and pages.

## Steps

1. **Initialize shadcn/ui**:
   ```bash
   bunx shadcn@latest init
   ```
   - Style: new-york
   - Base color: neutral
   - CSS variables: yes
   - CSS path: `src/styles/app.css`
   - Aliases: match current `components.json` (`@/components`, `@/lib/utils`, `@/components/ui`, `@/lib`, `@/hooks`)

2. **Verify `components.json`** matches the target config from the ADR

3. **Verify `src/lib/utils.ts`** — shadcn init may regenerate it. Ensure it still exports `cn()` with `clsx` + `tailwind-merge`

4. **Install required UI components** (matching what the Vue project uses):
   ```bash
   bunx shadcn@latest add button card tabs sidebar tooltip navigation-menu input alert skeleton separator label dialog sheet textarea collapsible breadcrumb sonner
   ```

5. **Verify** components are generated in `src/components/ui/` and import correctly

## Acceptance Criteria

- `components.json` is configured with new-york style and correct aliases
- All UI components listed above are installed in `src/components/ui/`
- `cn()` utility works
- A test route can render a `<Button>` component

## Estimated Scope

~20+ files generated by shadcn CLI, 1 config file
